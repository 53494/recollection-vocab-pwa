## 1. 文档目的与范围

本 PRD 汇总 Recollection 在生成项目原型前形成的产品构想，并与当前代码原型进行对照，明确：

## 2. 产品概述

### 2.1 产品定位

**Recollection** 是一款面向英语专业学生的本地优先、深度学习型英语词汇记忆 PWA。它不以“看过并认识单词”为终点，而是通过：

> **中文释义/语境提示 → 用户自主英文造句 → AI 诊断语法、搭配与地道表达 → 间隔复习与再次输出**

帮助用户把单词从识别记忆推进到能够在真实语境中正确使用。

### 2.2 用户痛点

传统背词工具通常擅长展示词义、播放发音和进行选择题，但对以下问题支持不足：

1. 用户知道中文意思，却不会在句子中使用单词；
2. 用户的语法、词性、搭配和语域错误缺少即时反馈；
3. 只重复“认不认识”的判断，难以形成主动输出；
4. 复习节奏容易依赖自觉，遗忘前缺少明确提醒和调度；
5. 学习数据、收藏例句和个人积累缺少统一沉淀；
6. 移动学习场景下网络不稳定，学习内容不应完全依赖在线服务。

### 2.3 产品价值

- **从识别到使用**：以中译英造句作为核心主动回忆任务；
- **反馈具体可执行**：不仅判对错，还解释语法、搭配和地道表达；
- **长期记忆导向**：用艾宾浩斯阶段调度新词、复习词和错词；
- **个人知识沉淀**：把收藏例句和 AI 建议积累为个人语料库；
- **本地优先**：词书、进度和学习记录本地保存，支持离线学习与数据备份。

## 3. 目标用户与使用场景

### 3.1 目标用户

**核心用户**：英语专业本科生，尤其是准备 TEM-4/TEM-8、希望提升词汇实际运用能力的学习者。

**用户特征**：

- 已具备基础英语阅读能力，但希望提高词汇的主动使用能力；
- 能接受中英文混合学习界面；
- 主要通过 Android 手机碎片化学习；
- 需要轻量、低干扰、可持续的学习工具；
- Demo 阶段默认是个人使用者，不要求注册和云端同步。

### 3.2 核心使用场景

| 场景 | 用户目标 | 产品支持 |
|---|---|---|
| 首次使用 | 快速开始，不被复杂配置阻塞 | 选择词书、设置每日目标、进入首页 |
| 日间学习 | 学会一批新词及其语境 | 单词卡、音标、发音、3-4 条例句、认识程度反馈 |
| 学后巩固 | 检查拼写并处理错词 | 拼写测试、错误自动循环重测 |
| 晚间复习 | 把词义转化为主动英文输出 | 复习本、中译英造句、AI 批改 |
| 零散自测 | 快速检查听辨、释义和拼写 | 听音辨意、填空拼写、错题重测 |
| 长期积累 | 保存可迁移的表达 | 收藏例句、积累本、AI 推荐一键保存 |
| 周期回顾 | 了解一周/一段时间的掌握情况 | 每周复习、统计仪表盘 |
| 离线学习 | 无网络时继续学词和复习 | IndexedDB 本地数据、PWA 静态资源缓存 |
| 个性化管理 | 调整应用和备份数据 | 中英文、日/夜间、主题色、每日目标、导入导出、反馈 |

## 4. 产品目标与成功标准

### 4.1 Demo 阶段目标

1. 验证“自主造句 + AI 诊断”是否比单纯选择题更有学习价值；
2. 打通“选词书 → 学习 → 拼写/自测 → 复习 → 积累 → 统计”的核心闭环；
3. 验证本地优先架构在 Android PWA 中可用；
4. 让个人测试者能够在没有账号和后端的情况下持续使用并备份数据。

### 4.2 可观察成功标准

- 新用户能够在首次进入后完成选词书和每日目标设置；
- 用户能够独立完成一轮单词学习，并看到学习进度与完成结果；
- 学习后的单词可以进入拼写测试、自测和到期复习；
- 收藏例句能够进入复习本，并完成中译英造句；
- 在线时 AI 能返回结构化诊断，失败时有明确提示且不丢失学习状态；
- AI 建议能够进入积累本，用户也能手动添加表达；
- 统计页面能展示学习趋势、掌握分布和连续学习情况；
- 刷新页面或重新打开 PWA 后，本地数据仍然存在；
- 用户可以导出 JSON 备份，并在确认后导入恢复。

### 4.3 暂不作为 Demo 成功标准

- 多用户账号、云端同步、社交排行榜；
- AI 诊断的专业级语言学准确率；
- 全量 TEM-4/TEM-8 词库；
- 每月播客的完整音频生成与分发能力。

## 5. 产品信息架构与导航

### 5.1 页面结构

- **启动/欢迎**：Splash、首次引导、词书选择、每日目标设置；
- **首页**：今日统计、开始学习、晚间复习、自测中心、积累本、功能引导；
- **词书**：词书列表及可用状态；
- **学习**：单词卡、例句、收藏、认识程度、拼写测试；
- **复习本**：拼写单词、造句练习、收藏/待复习句子；
- **复习会话**：中文提示、英文输入、AI 反馈、下一题/完成；
- **积累本**：AI 推荐与手动添加的表达；
- **自测中心**：听音辨意、填空拼写；
- **统计**：摘要、近 14 日学习图、掌握分布、近 12 周热力图；
- **周期复习**：每周复习；每月播客（规划/占位）；
- **设置**：主题、语言、反馈、每日目标、数据备份与清除、重新查看引导。

### 5.2 主要用户流程

```text
首次打开
  → Splash
  → 选择词书
  → 设置每日目标
  → 首页

首页
  → 开始学习
  → 查看单词/例句/发音
  → 收藏例句（可选）
  → 认识/模糊/不认识
  → 完成学习
  → 拼写测试
  → 错词循环重测

首页/复习本
  → 选择拼写或造句
  → 中文提示
  → 输入英文句子
  → AI 批改/查看参考原句
  → 添加到积累本（可选）
  → 完成复习

首页/自测中心
  → 听音辨意或填空拼写
  → 即时反馈
  → 结果及错题
  → 只测错题/全部重测
```

## 6. 功能需求

优先级定义：**P0 = 核心闭环必须具备；P1 = Demo 重要配套；P2 = 后续增强**。

### 6.1 首次进入与引导（P0）

**需求**

- 显示产品启动页，用户点击后进入首页或首次欢迎流程；
- 首次用户选择可用词书；无词数据的词书显示“即将上线”并不可选；
- 用户选择每日学习目标：10/20/30 或自定义数量；
- 确认后保存 onboarding 状态，后续直接进入首页；
- 首页首次展示分步功能引导，可跳过，设置中可重新查看。

**验收标准**

- 首次进入不会直接进入无词书的学习页；
- 完成设置后刷新仍保持已选词书和目标；
- 引导文案准确覆盖学习、复习、自测、统计和个性化设置。

### 6.2 词书系统（P0）

**需求**

- 提供分级词书元数据：初中、高中、CET-4、CET-6、TEM-4、TEM-8 等；
- 展示名称、中文名、简介、图标、词汇量和可用状态；
- 支持用户切换当前词书；
- 支持后续导入自定义词书；
- Demo 使用小规模种子数据验证流程，当前主要为 CET-4 30 词。

**业务规则**

- 只有词数大于 0 的词书可开始学习；
- 当前词书和每日目标由本地状态持久化；
- 词书和单词数据保存在本地数据库。

### 6.3 单词学习与拼写（P0）

**单词卡必须展示**

- 英文单词、IPA 音标、词性、中文释义；
- 每词 3-4 条不同语境例句，展示英文及中文翻译；
- 点击按钮使用 Web Speech API 播放发音；
- 例句可收藏/取消收藏，收藏后进入复习本。

**学习反馈**

- “认识”：质量 5，艾宾浩斯阶段推进；
- “模糊”：质量 3，阶段保持；
- “不认识”：质量 1，阶段回退；
- 更新单词状态、复习次数、正确/错误次数、记忆保留率和下次复习日期；
- 显示当前进度和完成结果。

**拼写测试**

- 以中文释义和发音为提示输入英文拼写；
- 即时判断拼写；
- 错词自动加入本轮重测，直至完成；
- 展示正确数、总数和正确率。

### 6.4 艾宾浩斯复习调度（P0）

复习阶段固定为：

```text
首次学习 → 20 分钟 → 1 小时 → 1 天 → 2 天 → 6 天 → 14 天 → 30 天 → 已掌握
```

**规则**

- 质量 4-5：阶段 +1，最高到 7；
- 质量 2-3：阶段保持；
- 质量 0-1：阶段 -1，最低保持在阶段 1；
- 阶段 7 标记为已掌握；
- 到期词可进入学习补充、晚间复习和自测池；
- 记录理论保留率与用户表现，用于统计和后续优化。

### 6.5 复习本与 AI 造句批改（P0）

**复习本入口**

- 提供“拼写单词”和“造句练习”两种模式；
- 拼写模式使用今日已学/到期单词，错词循环重测；
- 造句模式优先使用用户收藏例句；没有收藏时，允许从今日单词中选择不熟悉词或全部单词。

**造句交互**

- 显示中文句子，不直接显示英文原句；
- 引导用户使用目标单词自主写英文句子；
- 支持跳过并查看参考原句；
- 提交前拦截空输入；
- AI 请求期间显示提交中状态，禁止重复提交。

**AI 反馈内容**

AI 返回结构化结果，至少包含：

1. 总体评价与 0-100 分；
2. 目标词是否使用、使用形式、拼写及可替换同义词；
3. 语法问题：原文、改写、中文解释；
4. 搭配问题：不自然表达、地道替换、解释；
5. 参考原句中的习语、搭配和地道表达；
6. 可添加到积累本的推荐表达。

**异常规则**

- AI 服务未配置、网络失败、接口非 2xx、返回为空或 JSON 格式异常时，显示可理解的错误；
- AI 失败不应破坏已保存的学习进度；
- Demo 阶段通过受控 Cloudflare Worker 调用 OpenAI 兼容接口，浏览器不保存或接触上游 API Key。

### 6.6 积累本（P1）

- 保存 AI 推荐表达或用户手动添加表达；
- 每条记录包含表达、语境例句、中文释义、标签、来源和添加时间；
- 来源区分“AI 推荐”和“手动添加”；
- 支持空状态、列表浏览和手动添加表单；
- 后续可增加标签筛选、编辑、删除、复习次数和间隔复习。

### 6.7 自测中心（P1）

#### 听音辨意

- 从今日已学或到期词中抽题；
- 自动/手动播放英文发音；
- 从 4 个中文释义中选择正确项；
- 可切换显示/隐藏英文拼写调整难度；
- 即时展示正确/错误颜色反馈；
- 结束后显示正确率、错题列表，支持只测错题或全部重测；
- 保存自测记录并更新每日统计。

#### 填空拼写

- 展示例句并将目标词挖空，附中文释义；
- 优先使用收藏句，否则随机使用例句；
- 输入目标词并进行大小写不敏感比对；
- 展示用户答案与正确答案；
- 保存测试记录，结束后支持错题重测。

### 6.8 学习统计仪表盘（P1）

展示：

- 连续打卡天数、活跃天数、已学习单词、学习中/已掌握数量；
- 最近 14 天新学、复习、自测数据；
- new / learning / reviewing / mastered 掌握分布；
- 最近 12 周学习热力图；
- 后续可增加累计学习时长、平均正确率和理论/实际遗忘曲线对比。

### 6.9 每周复习（P1）

- 汇总本周学习过的单词；
- 第一阶段：单词与中文释义选择，检验识别；
- 第二阶段：中文释义/发音提示下拼写；
- 分别展示认识和拼写结果；
- 错词可循环重测；
- 显示每周复习结果。

### 6.10 每月播客（P2 / 后续实现）

规划目标：

- 汇总本月需要复习的单词；
- 每词播放 2-3 遍，词间约 3 秒间隔；
- 支持后台/连续播放、暂停和结束状态；
- 当前原型仅保留入口和占位页，不作为 Demo 核心验收项。

### 6.11 设置、反馈与数据管理（P1）

**个性化**

- 中文/英文界面切换；
- 日间/夜间模式；
- 自定义强调色和启动页底色；
- 修改每日学习目标；
- 首页功能引导重新查看。

**反馈**

- 查看版本信息；
- 提交产品反馈；
- Demo 阶段反馈保存在本地，不承诺服务端收集。

**数据**

- 导出词书、单词、学习进度、复习记录、积累本、自测记录和每日日志为 JSON；
- 导入前必须明确提示会覆盖当前数据；
- 导入格式错误时给出错误提示；
- 提供清除全部本地数据，并明确“不可恢复”确认。

> 注：历史需求曾包含“管理 API Key”入口；当前产品文案和设置体验已将其移除。AI 配置由受控服务端代理管理，不在首页个性化设置引导中宣传。

### 6.12 PWA 与离线（P1）

- 支持 Android 添加到桌面和 standalone 模式；
- 缓存 HTML、JS、CSS、图标等静态资源；
- 词书、学习进度和统计数据离线可读写；
- 无网络时仍可完成不依赖 AI 的学习、拼写和自测；
- AI 相关操作在无网络时显示明确提示，后续可扩展待处理队列；
- GitHub Pages 子路径部署需保持正确的 Vite base 和 Hash 路由。

## 7. 数据与业务规则

### 7.1 核心实体

| 实体 | 用途 |
|---|---|
| WordBook | 词书元数据与排序 |
| Word | 单词、音标、词性、释义、例句 |
| LearningProgress | 单词状态、复习阶段、正确率、下次复习 |
| ReviewEntry | 用户收藏的例句 |
| ReviewAttempt | 造句提交、AI 反馈、得分 |
| ReviewSession | 日/周/月复习会话汇总 |
| AccumulationEntry | AI 推荐或手动保存的表达 |
| SelfTestAttempt | 听音辨意/填空测试记录 |
| DailyLog | 每日学习量、自测量、复习量和打卡 |
| Settings | 可持久化的应用配置 |

### 7.2 本地优先原则

- 默认不建立账号，不依赖业务后端；
- IndexedDB 保存学习数据，localStorage 保存轻量偏好和引导状态；
- 数据库访问应通过数据服务层封装，避免页面直接散落数据库逻辑；
- 删除和导入均属于高影响操作，必须二次确认或明确警告。

### 7.3 统计口径

- “已学”以单词学习操作写入每日日志为准；
- “复习”以造句提交写入每日日志为准；
- “自测”以完成测试写入每日日志为准；
- 连续打卡由连续存在有效学习活动的日期计算；
- 掌握分布按 LearningProgress.status 统计。

## 8. 非功能需求

### 8.1 视觉与交互

- 采用黑、白、灰极简视觉体系；
- 黄色 `#F59E0B` 仅用于目标词高亮及必要的单词语义强调，不作为泛化装饰色；
- 以移动端 375px 为基准，内容最大宽度约 512px；
- 触摸目标不小于 44×44px；
- 卡片圆角、留白、进度条和反馈动画保持统一；
- 正确/错误反馈应即时、短促且有文字替代；
- 文字与背景对比度达到 WCAG AA（正文至少 4.5:1）；
- 界面支持响应式适配手机与桌面浏览器。

### 8.2 性能

- 首屏目标加载时间小于 3 秒，PWA 缓存后目标小于 1 秒；
- 常规 IndexedDB 查询目标小于 50ms；
- AI 请求目标响应时间小于 5 秒，超时或失败需可恢复；
- 页面按路由懒加载，避免首次加载所有页面资源。

### 8.3 可靠性与容错

- 所有异步操作提供 loading、error、空状态和重试/返回路径；
- 数据库初始化失败时展示错误和重试按钮；
- 空输入、空词书、无到期词、无收藏句等边界状态有明确引导；
- AI 返回必须做字段兜底，异常 JSON 不得导致页面崩溃；
- 重复提交、重复点击和自动跳题需要有防抖/禁用机制。

### 8.4 隐私与数据安全

- 学习数据默认只保存在用户设备；
- 不收集账号、学习内容或行为分析数据；
- 上游 AI API Key 只保存在 Cloudflare Worker Secret 中，不写入仓库、前端环境文件或构建产物；
- 导出文件由用户主动下载，导入和清除前明确告知影响；
- 浏览器只调用受 CORS、输入校验、限流与超时保护的服务端代理。

### 8.5 可维护性

- React + TypeScript 严格模式；
- 组件保持单一职责；
- 数据库访问由 `dataService.ts` 统一封装；
- 业务类型集中维护，避免隐式 `any`；
- 每个 Phase 完成后进行构建、类型检查和人工流程验收。

- [ ] 单词卡信息完整，发音可用；
