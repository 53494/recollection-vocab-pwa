# 开发指引 - Recollection

## 工作流程

### 开发模式
- **逐步推进**：严格按照 Phase 1 → 2 → 3 → ... → 10 的顺序执行
- **每步确认**：每个 Phase 完成后**必须先让用户审核**，用户确认后再推进下一步
- **不要一次性生成所有代码**：每个 Phase 结束后停下来，等待用户决策

### Phase 概览

| Phase | 内容 | 预计时间 |
|-------|------|----------|
| 1 | 项目搭建 + 基础框架 | 1 天 |
| 2 | 数据库 + 类型 + 种子数据 | 2 天 |
| 3 | 词书 + 学习流程 | 3 天 |
| 4 | 复习本 + AI 集成 | 4 天 |
| 5 | 积累本 | 1.5 天 |
| 6 | 首页 + 艾宾浩斯调度器 | 2 天 |
| 6.5 | 自测中心 | 2.5 天 |
| 6.6 | 学习统计仪表盘 | 2 天 |
| 7 | 每周复习 | 2 天 |
| 8 | 每月播客 | 1.5 天 |
| 9 | PWA 离线 + 安装 | 2 天 |
| 10 | 设置 + 数据管理 | 1.5 天 |

### 每个 Phase 的标准流程

1. **开发前**：确认上一 Phase 已完成并验收
2. **开发中**：按照该 Phase 的步骤列表逐项实现
3. **开发后**：执行该 Phase 的验证项，确保功能正常
4. **汇报**：向用户展示完成的内容和验证结果
5. **等待确认**：用户说"可以进行下一步"后再开始下一个 Phase

### 质量标准
- TypeScript 严格模式，无 `any` 类型（除非有充分理由）
- 组件遵循单一职责原则
- 数据库操作使用 `dataService.ts` 封装，不在组件中直接操作 Dexie
- UI 严格遵循 `design-specs.md` 的色彩和间距规范
- 所有异步操作有 loading/error 状态处理
